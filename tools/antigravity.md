根据提供的来源，视频详细介绍了如何在 **Antigravity IDE** 中使用、创建及调用 **Agent Skills** 的具体操作步骤。以下是整理后的操作指南：

### 一、 使用现有的 Agent Skills（以官方技能为例）
1.  **环境准备：** 确保 Antigravity IDE 已升级至**最新版本**。
2.  **克隆项目：** 使用 `git clone` 命令将官方发布的 Agent Skills 项目克隆到本地。
3.  **放置技能文件：** 根据需求选择存放路径：
    *   **项目级（局部）：** 放在当前项目的路径下，仅供该项目调用。
    *   **全局级：** 放在官方推荐的**全局路径**下，使所有项目都能调用。可以通过命令行移动文件，也可以手动将技能文件**复制粘贴**到 Antigravity 的 skills 存放路径中。
4.  **调用与执行：**
    *   在 Antigravity 中输入提示词，并明确指定要使用的技能（例如：“创建咖啡店落地页并使用前端设计技能”）。
    *   **授权运行：** 当 IDE 提示需要运行命令（如调用模型生成图像）时，点击**允许执行**。
    *   **查看结果：** 复制 IDE 生成的运行命令（如本地服务器命令）并在终端执行，点击生成的链接即可预览效果。

### 二、 手动创建一个新的 Skill
1.  **创建路径：** 在当前项目路径下，通过终端执行命令创建一个专门用于存放 Agent Skills 的**文件夹**。
2.  **编写技能文件：** 在该路径下新建一个名为 **`skill.md`** 的文件。
3.  **定义内容：** 将工作流、最佳实践或脚本工具以 Markdown 格式写入该文件并保存。
4.  **测试调用：** 在对话框中输入指令调用该技能（例如：“使用 code review 技能审计当前项目代码”），IDE 会自动读取该 `.md` 文件并执行分析。

### 三、 安装并使用高级技能（以 UIUX Pro Max 为例）
1.  **安装项目：** 在 Antigravity 的终端中执行 `npm` 安装命令来获取该技能项目。
2.  **针对性配置：** 执行针对 Antigravity 的特定安装命令，确保技能被正确集成。
3.  **唤起技能：** 在输入框使用 **“/”** 快捷命令调出 **UIUX Pro Max** 技能界面。
4.  **输入具体需求：** 描述你想要构建的产品（例如：“使用 React 构建一个泥物化风格的 Todo List”），AI 将结合技能中的最佳实践生成美观且具有现代感的 UI。

### 补充工具
*   如果你需要将任何开源项目或网站转化为 Agent Skill，可以使用开源项目 **Skill Seeker** 进行一键转换。

**总结：** 
Agent Skills 的核心在于将临时指令（Prompt）转化为**长期资产**。通过上述步骤，用户可以将特定的业务手册、工作流或脚本打包，让 AI 助手能够按需加载并稳定产出符合标准的成果。

根据视频来源，这套“Vibe Coding”开发流程可以总结为从想法到上线的**五个核心步骤**。以下是详细的操作摘录：

### 第一步：使用 Stitch 设计 UI 界面
1.  **初始生成**：在 **Stitch** 中选择设计移动端或 PC 端应用，选择 AI 模型（如 Gemini 3.5），输入提示词生成初始 UI 页面。
2.  **隐私设置**：在 `Settings` 中可查看积分额度，并根据需要取消勾选“数据用于训练”以保护隐私。
3.  **UI 调整与优化**：
    *   **语言修改**：全选页面，要求 AI 将文字改为简体中文并设定字体（如思源黑体）。
    *   **精准编辑**：使用 `Annotate`（注释）功能框选特定区域（如宠物分类），输入指令删除或修改特定内容。
    *   **主题设定**：使用 `Edit Theme` 调整配色（如橙黄色）、模式（白天/黑夜）及圆角字型。
4.  **导出与协作**：查看 HTML 代码，或通过插件（如 Stitch code to Figma）将设计稿导出至 **Figma** 进行二次微调。

### 第二步：使用 AI Studio 生成交互式前端
1.  **导入与整合**：在 Stitch 中全选页面并点击 `Export as Studio`，系统会自动跳转至 **AI Studio**。
2.  **建立交互逻辑**：点击执行让 AI 将独立页面整合为具备点击响应和页面跳转功能的前端 App。
3.  **功能修补**：通过对话或画笔工具（添加箭头、框选）指示 AI 修复问题，例如添加“提交申请”按钮或完善消息通知功能。
4.  **下载代码**：确认前端功能无误后，将前端代码下载至本地。

### 第三步：使用 Antigravity 生成后端与数据库
1.  **代码分析**：将前端代码导入 **Antigravity**，要求 AI 分析并生成对应的后端接口、数据逻辑及数据库结构（指定使用 **Supabase** 在线数据库）。
2.  **数据库配置**：
    *   在 **Supabase** 新建项目，获取 API Key 和 URL。
    *   将参数填入项目的 `.env` 配置文件中。
3.  **初始化表结构**：在 Supabase 的 `SQL Editor` 中运行 AI 生成的 SQL 语句，建立宠物信息、领养申请、用户信息等数据表。
4.  **本地调试**：执行 `npm run dev` 启动服务，测试用户注册、登录及提交领养申请等核心流程，确保数据能正确存入数据库。

### 第四步：将代码托管至 GitHub
1.  **仓库初始化**：在 AI 的引导下执行 `git init`、`git add` 和 `git commit` 指令。
2.  **远程推送**：在 GitHub 新建项目并复制 SSH 网址，在 Antigravity 中提供给 AI，由其自动完成代码推送（Push）。

### 第五步：使用 Vercel 一键部署上线
1.  **项目导入**：登录 **Vercel**，安装 GitHub 插件并导入对应的项目仓库。
2.  **部署配置**：
    *   设定构建命令（如 `npm run build`）和安装依赖命令（如 `npm install`）。
    *   **环境变量**：将数据库的 API 参数配置到 Vercel 的 Environment Variables 中。
3.  **正式发布**：点击 `Deploy`，部署成功后 Vercel 会分配一个外网访问网址，至此 App 正式上线。

**理解这套流程的类比：**
这套流程就像是**全自动化的建筑工程**。**Stitch** 是负责画出精美图纸的**设计师**；**AI Studio** 是负责把图纸变成实体样板间的**装修队**；**Antigravity** 则是负责铺设隐藏管线（后端逻辑）和挖掘地窖（数据库）的**工程师**。最后，**GitHub** 相当于一份**施工档案库**，而 **Vercel** 则是帮你把这栋房子从私人领地搬到**闹市中心（云端）**，让所有人都能进去参观和使用。

根据提供的视频来源，该视频演示了如何使用 AI 程式码编辑器 **Google Antigravity**，通过 "Vibe Coding" 的方式完成一个连接 Google Sheet 作为资料库的全端专案（闯关游戏）。

以下是摘录自视频的操作步骤：

### 一、 环境准备与基础配置
1.  **安装环境**：如果专案涉及 React 框架，需先安装 **Node.js**。
2.  **启动编辑器**：下载并安装 Google Antigravity（支援 Windows, Mac, Linux），并使用 Google 账号登录。
3.  **建立工作区**：新建一个专案资料夹，并将其拖入 Antigravity 中，让 AI 识别工作目录。
4.  **设定 AI 规则 (Customizations)**：
    *   点击右侧设定中的 `customizations`，在 **Global Rules** 中贴入角色设定（如：资深前端工程师与游戏设计师）。
    *   增加语言偏好规则，例如贴上 "Response in Traditional Chinese"（以繁体中文回复）并点击刷新。

### 二、 专案规划与初步开发
1.  **选择模型与模式**：
    *   在对话窗选择模型（如 Gemini, Claude 或 GPT 系列）。
    *   处理复杂任务（如初始规划）时选择 **`planning`** 模式；简单问答则选 `fast` 模式。
2.  **发送精准需求**：在对话框输入包含**角色设定、技术栈（React + Google Apps Script + Google Sheet）、设计风格（像素风）、逻辑规范及资料库栏位配置**的详细提示词。
3.  **执行任务**：观察 AI 在左侧规划的任务进度，并在 AI 完成初步程式码初始化后，点击 **`Step all`** 以接受并应用所有调整。

### 三、 模拟测试与迭代修正
1.  **开启模拟执行**：输入指令“请开启浏览器，模拟执行并检查是否有错误”。
2.  **UI/UX 调整**：针对 AI 自动开启的浏览器画面进行检查，若发现 RWD（响应式设计）破板或逻辑问题，直接在对话框向 AI 反馈修正。
3.  **生成辅助文档**：请 AI 撰写 `README.md` 操作说明文件，并生成测试用的选择题数据（以表格形式呈现）。

### 四、 串接 Google Sheet 资料库
1.  **建立云端表格**：在 Google Sheet 中根据 AI 提供的栏位名称，分别建立“题目”和“回答”两个工作表。
2.  **配置 Google Apps Script**：
    *   在 Google Sheet 中点击“扩展功能” -> “Apps Script”。
    *   从 Antigravity 中复制 `code.gs` 的完整内容，贴入 Apps Script 编辑器。
    *   点击“部署” -> “新增部署”，类型选择“网页应用程式”，设定执行身份为“我”，存取权限为“所有人”。
    *   授权 Gmail 存取权限（若遇警告需点选“进阶/Advanced”并继续执行）。
3.  **设定环境变数**：复制部署后的网页应用程式 URL，贴回专案中的 `.env` 档（或 `.env.example`）。

### 五、 部署至 GitHub Pages
1.  **建立 GitHub Action**：请 AI 设计自动部署至 GitHub Pages 的工作流文件（workflow），并更新相关操作流程至 `README.md`。
2.  **版本控制与推送**：
    *   点击左侧 `Source Control`，初始化仓库 (`Initialize Repository`)。
    *   点击 `Generate` 让 AI 生成 Commit 信息，随后点击 `Commit` 记录变更。
    *   点击 `Publish Branch` 将专案发布为 **Public Repository** 并上传至 GitHub。
3.  **GitHub 线上设定**：
    *   在 GitHub 仓库的 `Settings` -> `Pages` 中，将来源设定为 `GitHub Actions`（或选择对应的分支）。
    *   在 `Settings` -> `Secrets and variables` -> `Actions` 中新增 **Repository Secrets**，填入 `.env` 中的敏感资讯（如 Google Apps Script URL）。
4.  **最终触发**：如果部署后画面空白或无法读取资料，检查 `Actions` 分页的部署状态，必要时手动点击 `Re-run all jobs` 以确保环境变数生效。
