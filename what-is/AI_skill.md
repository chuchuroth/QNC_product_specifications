根据视频来源提供的信息，OpenCode 是一款开源的 AI 编程工具，具备与 Claude Code 类似的各项功能，并支持多种运行形态和模型配置。以下是视频中提取的具体操作步骤：

### 一、 安装 OpenCode
OpenCode 提供了四种形态，以下是主要的安装步骤：
1.  **命令行版（CLI）安装**：
    *   访问 Node.js 首页，根据操作系统下载并安装 **Node.js**。
    *   复制 npm 安装命令，在终端窗口中运行以完成安装。
    *   输入命令 `open-code` 即可启动并进行初始配置。
2.  **桌面客户端安装**：
    *   下载客户端安装包并完成安装。
    *   打开后选择一个文件夹作为**项目文件夹**即可开始使用（注：目前处于 beta 测试版，功能较基础）。
3.  **VS Code 插件版安装**：
    *   前提是已安装命令行版本。
    *   在 VS Code 的 Extensions 搜索并安装 **open-code** 插件。
    *   使用快捷键 `Ctrl/Shift + P` 打开命令面板，输入 `open-code` 以插件形式启动。
    *   使用快捷键 `Alt + K` 可以将选中的代码快速粘贴到聊天窗口。

### 二、 模型配置与接入
1.  **使用内置免费模型**：
    *   在软件中输入命令 `/models`，带有 `free` 标记的模型（如 Gemini 1.5 Pro, Minimax 2.1）可零配置直接使用。
2.  **接入顶级模型（通过 Antigravity OS 插件）**：
    *   在 GitHub 首页复制安装提示词，粘贴到 OpenCode 中让 AI 自动完成安装。
    *   打开新终端执行登录命令，选择 **Google** 作为供应商并登录谷歌账户。
    *   完成配置后重启，即可通过 `/models` 选择 **Gemini 1.5 Pro** 或 **Claude 3.5 Opus**。
3.  **接入 GPT-4o (OpenAI)**：
    *   输入命令 `/login` 并选择 OpenAI 的 **GPT Pro** 选项。
    *   在浏览器中打开链接并选择继续以完成登录。
4.  **接入 OpenRouter**：
    *   在 OpenRouter 官网获取 **API Key**。
    *   在 OpenCode 中输入该 Key，即可接入市面上几乎所有的大模型。

### 三、 高级功能操作
1.  **Session（会话）管理**：
    *   输入 `new` 命令可以开启一个新的 Session，实现**多任务并行开发**。
    *   输入 `/sess` 可以查看并切换正在后台运行的会话。
2.  **分享与历史记录**：
    *   输入 `/share` 可将对话记录生成网页链接分享给他人。
    *   输入 `/export` 可将对话记录导出为文件。
    *   输入 `/timeline` 可查看时间线快照，并使用 **revert** 功能将代码和聊天回滚到之前的状态。
3.  **迁移 Agent Skills**：
    *   在项目文件夹中新建 `.opencode` 文件夹，并在其下新建 `skills` 文件夹。
    *   将原有的技能文件夹复制进来，并将目录中的 `.claude-code` 替换为 `.opencode`。
4.  **配置 MCP（模型上下文协议）**：
    *   **本地 MCP**：在配置文件 `opencode.json` 中添加 MCP 段落，设置类型为 `local` 并填写对应的 `npx` 启动命令。
    *   **远程 MCP**：在配置文件中设置类型为 `remote`，并填入服务器的 **URL** 和 **API Key**。

### 四、 自动化与云端运行
1.  **Ultraw 模式（智能体调度）**：
    *   安装相关插件后，输入魔法词 `w` 或 `ultraw`。
    *   AI 会自动将任务拆分为待办清单（To-do list），并调度多个子智能体（如前端工程师、先知等）并行工作。
2.  **GitHub 终端集成**：
    *   将项目上传至 GitHub 公开仓库。
    *   在本地执行安装命令，并按照提示将生成的配置文件提交到 GitHub。
    *   在 GitHub 项目的 **Settings > Secrets and variables > Actions** 中添加必要的 API 密钥。
    *   在 GitHub Issue 中评论 `/open-code`，即可触发 **GitHub Actions** 自动进行代码修复和创建 PR。

### 五、 自定义指令与智能体
1.  **自定义命令**：在配置文件的 `command` 文件夹下新建 `.md` 文件，定义命令名称、模式（Build/Plan）和描述，即可通过 `/` 调用。
2.  **自定义智能体**：在 `agent` 文件夹下新建 `.md` 文件，指定其为 **Primary**（主智能体，按 Tab 键切换）或 **Sub-agent**（子智能体，由系统自动调度）。

根据提供的视频来源，Codex 是一款功能强大的 AI 编程工具。以下是从视频中提取的具体操作步骤：

### 一、 CLI（命令行）环境安装与配置

2.  **安装 Codex**：在终端窗口运行从 Codex GitHub 页面复制的安装命令。
3.  **启动与登录**：在项目文件夹右键打开终端，输入 `codex` 启动。此时会弹出浏览器页面，需登录 ChatGPT 账户（建议使用 GPT Plus 账户以获得最佳效果）。
4.  **模型切换**：使用命令 `/model` 可在通用模型 (GPT-4o) 与编程专用模型 (GPT-4o-Codex) 之间切换，并可选择 L（快）、M（中）、H（高推理）三个档位。

### 二、 接入免费国产模型（以魔搭社区为例）
1.  **获取 API Key**：在魔搭社区注册并新建“访问令牌”，并在模型库中筛选支持推理 API 的模型（如 Qwen 2.5），获取其 **Base URL** 和 **模型名称**。
2.  **修改配置文件**：打开用户目录下的 `.codex` 文件夹（Windows 通常在 `C:\Users\用户名\.codex`），编辑 **`config.toml`** 文件。
3.  **设置变量**：在 `config.toml` 中填入模型名和 URL。API Key 不能直接写入文件，需先在系统的“**环境变量**”中新建一个变量，填入 Key 值，然后在配置文件中引用该变量名。
4.  **重启工具**：保存文件后重新启动 Codex 即可免登录使用国产模型。

### 三、 核心命令操作
*   **项目初始化**：输入 `/init`，Codex 会通读当前文件夹并生成 `context.md` 文件，帮助 AI 理解项目背景。
*   **管理对话**：
    *   `/compact`：压缩对话上下文，提高 AI 专注度并降低消耗。
    *   `/new`：开启新对话并清除之前的记录。
*   **调整权限**：输入 `/approvals` 可切换为手动模式、自动模式或 **Full Access**（全自动执行命令和修改文件）。

### 四、 进阶功能：MCP 与自定义命令
1.  **安装 MCP 工具**：
    *   在 `config.toml` 中添加 MCP Server 配置。
    *   **注意**：Windows 用户需使用特定格式（如使用 `cmd /C`），Python 类的 MCP 则需使用 `uvx` 命令启动。
    *   输入 `/mcp` 查看已安装的工具。
2.  **创建自定义命令**：
    *   在 `.codex` 文件夹下新建 `prompts` 文件夹。
    *   在此文件夹内新建 `.md` 文件（文件名即为命令名），并在其中定义指令和参数（如使用 `$1`, `$2`）。

### 五、 IDE 插件版操作 (VS Code)
1.  **安装插件**：在扩展市场搜索并安装 **Open AAA**（Codex 官方插件）。
2.  **使用优势**：插件版与 CLI 版配置互通。它支持**可视化代码对比**、一键**撤回 (Undo)** 更改，并允许直接在对话框中**粘贴图片**。

### 六、 云端运行环境
1.  **绑定账号**：在 ChatGPT 网页版选择 Codex，并绑定 **GitHub** 账户。
2.  **上传项目**：通过 VS Code 的 Source Control 将本地项目发布到 GitHub 私有仓库。
3.  **异步执行**：在网页端选择对应的仓库，输入修改需求。AI 会在云端自动创建环境并修改代码。
4.  **提交更改**：审核代码无误后，点击“**创建拉取请求 (PR)**”提交到 GitHub，之后在本地点击 **Sync** 同步代码。
5.  **开启代码审计**：在网页版设置中打开“启动代码审核”，Codex 会对所有 PR 自动进行 **Code Review** 并提出建议。

### 七、 SDK 环境配置
1.  在文件夹中执行 `npm init -y` 初始化工程。
2.  执行 `npm install codex-sdk` 安装开发包。
3.  新建 `.js` 文件，粘贴官方示例代码，通过 `node` 命令运行，即可在自己的程序中调用 Codex 能力。

### 二、 接入免费模型 API (使用 Claude Code Router)
1.  **安装 Router 工具**：在命令行执行 `cloud-code-router` 的安装命令。
2.  **创建配置文件**：
    *   前往路径 `C:\Users\您的用户名\`，新建文件夹 `.cloud-code-router`。
    *   在该文件夹内新建配置文件 `config.json`。
    *   从项目页面复制配置模板并粘贴到该文件中。
3.  **获取并配置模型 API Key**：
    *   **千问 (ModelScope)**：在魔搭社区注册并绑定阿里云账号，在“访问令牌”中新建并复制 API Key。
    *   **Gemini (Google AI Studio)**：在 Google AI Studio 申请 API Key，并在配置文件中填入模型名称（如 `gemini-1.5-pro`）。
    *   **GLM-4.5 (智谱 AI)**：在智谱 AI 开放平台申请 API Key，配置提供商类型为 `openrouter` 并填入模型名 `glm-4.5`。
    *   **原生 Claude (OpenRouter)**：在 OpenRouter 充值并创建 API Key，在配置中对接到 `claude-3-5-sonnet`。
4.  **运行与调试**：
    *   在项目文件夹打开终端，输入 `ccr code` 启动。
    *   若启动失败（如端口占用），可在 `config.json` 中自定义 `port`（端口号）。
    *   修改配置后，需执行 `ccr stop` 停止服务后再重新启动以使配置生效。

### 三、 核心操作命令与快捷键
*   **`/init` (项目初始化)**：执行后 AI 会通读整个项目，并将生成的知识保存到 `CLAUDE.md` 文件中，方便 AI 快速了解背景。
*   **`/compact` (上下文压缩)**：压缩之前的对话记录，有效减少 Token 消耗。
*   **`/clear` (清除记录)**：开启新任务前清除之前的上下文，避免干扰。
*   **`Alt + M` (模式切换)**：在三种模式间切换：
    *   **普通模式**：所有代码修改需人工审核。
    *   **Auto-accept**：代码修改不再需要人工审批。
    *   **Plan Mode**：AI 只提供修改计划和想法，不实际动手改代码。

### 四、 高级功能操作
1.  **图像分析**：如果使用的模型（如 Gemini）支持识图，将网页截图保存到本地，然后直接**拖入对话窗口**并输入需求。

    *   在 VS Code 插件市场搜索并安装 **Claude Code** 插件。

根据视频来源（《AI编程工具Claude Code详细攻略》），以下是提取的操作步骤：

### 一、 基础安装与启动
1.  **安装环境**：如果电脑未安装 Node.js，需先访问官网进行安装。
2.  **安装工具**：在命令行窗口复制并运行 Claude Code 的安装命令。
3.  **启动方式**：
    *   **官方用户**：进入项目文件夹，输入 `claude` 并登录官网账号。
    *   **API/国内用户**：使用 **Claude Code Router (CCR)**，输入 `ccr code` 启动。

### 二、 项目管理与对话控制
1.  **项目初始化**：输入 `/init` 让 AI 通读项目文件并生成 `CLAUDE.md` 上下文文件。
2.  **上下文管理**：
    *   使用 `/compact` 压缩对话，排除无关内容以降低 Token 消耗。
    *   使用 `/clear` 清除历史记录，开启新任务。
3.  **控制思考强度**：在指令前添加 `think`、`harder`、`best` 或 `ultrac` 来增加 AI 的推理长度。
4.  **快捷模式**：
    *   **命令行模式**：输入 `!` 运行临时命令（如 `!npm install`），结果会自动加入对话上下文。
    *   **记忆模式**：输入 `#` 进入记忆模式，将信息（如版本号）保存为项目级或用户级的长期记忆。
5.  **非交互模式**：使用 `claude -p "指令"` 进行一次性快速调用。

### 三、 IDE 与外部集成
1.  **VS Code 打通**：
    *   在 VS Code 扩展市场安装 **Claude Code** 插件。
    *   在终端输入 `/ide vscode` 完成关联，实现代码选中感知和可视化差异对比。
2.  **GitHub 集成**：
    *   安装 **GitHub CLI** 工具并登录。
    *   要求 AI 读取 Issue（如 `gh issue view`）、修复代码、创建分支并推送到 GitHub。

### 四、 MCP（模型上下文协议）配置
1.  **安装 MCP**：退出 Claude Code，执行 `claudemcp add <名称> -- <启动命令>`（例如使用 `npx` 启动 Cex7）。
2.  **管理 MCP**：
    *   输入 `/mcp` 查看已安装的服务。
    *   使用 `claudemcp remove <名称>` 删除服务。
3.  **权限控制**：输入 `permissions` 设置 `allow`（自动允许）或 `deny`（禁止）规则。若需全自动运行，启动时加参数 `--dangerously-allow-permissions`。

### 五、 高级定制化功能
1.  **自定义命令**：在项目 `.claude/commands` 文件夹下新建 `.md` 文件，使用自然语言定义命令（支持 `$1` 等参数），之后通过 `/文件名` 调用。
2.  **自动化钩子 (Hooks)**：在 `.claude/settings.json` 中配置 `hooks`（如 `post-tool-use`），让 AI 在修改代码后自动执行格式检查等任务。
3.  **子智能体 (Subagent)**：输入 `/subagent create` 创建专注特定任务的小型智能体（如代码审核、天气查询），实现多任务并行处理。

### 六、 历史回退与可视化应用 (Cloudia)
1.  **对话回退**：输入 `/resume` 找回历史话题，双击 `ESC` 可跳转到特定对话点继续。
2.  **可视化客户端 (Cloudia)**：
    *   下载并安装社区提供的 **Cloudia** 安装包。
    *   若使用 CCR，需在设置中配置 `anthropic_token` 和 `anthropic_base_url`。
    *   **时间线导航**：点击 `create checkpoint` 创建检查点。若需还原，点击 `restore all` 即可同时回退代码改动和对话历史。

根据视频来源提供的信息，这套 Docker 实战教程涵盖了从安装、基础操作、自定义镜像到多容器编排的全流程。以下是提取的具体操作步骤：

### 一、 安装 Docker
Docker 在不同操作系统上的安装流程如下：
*   **Linux (以 Ubuntu 为例)**：
    1.  访问 `get.docker.com` 获取安装脚本。
    2.  在终端执行脚本的第一步和第四步命令完成安装。
    3.  若非 root 用户，执行所有 `docker` 命令时需在前面添加 `sudo`。
*   **Windows**：
    1.  在“启用或关闭 Windows 功能”中勾选 **Virtual Machine Platform** 和 **WSL** (适用于 Linux 的 Windows 子系统)。
    2.  重启电脑后，在管理员模式的终端执行 `wsl --set-default-version 2` 并更新 WSL。
    3.  从官网下载并安装 **Docker Desktop** (建议根据 CPU 架构选择安装包)。
*   **Mac**：根据芯片类型（Intel 或 M 系列）下载对应的安装包并直接安装。

### 二、 基础配置与网络优化
由于网络环境问题，通常需要配置**镜像站**：
1.  **Linux**：修改配置文件（通常位于 `/etc/docker/daemon.json`），加入 `registry-mirrors` 列表，保存后使用命令重启 Docker 服务。
2.  **桌面版 (Win/Mac)**：在 Docker Desktop 的 **Settings > Docker Engine** 中修改 JSON 配置文件，点击 **Apply and restart**。

### 三、 镜像 (Image) 管理
*   **下载镜像**：使用 `docker pull [镜像名:标签]`。镜像完整名称由“注册表地址/命名空间/镜像库名:标签”组成，官方镜像可省略地址和命名空间。
*   **查看与删除**：
    *   `docker images`：列出本地所有镜像。
    *   `docker rmi [镜像ID或名称]`：删除特定镜像。

### 四、 容器 (Container) 运行与操作
*   **创建并运行**：使用 `docker run [参数] [镜像名]`。
    *   **核心参数**：
        *   `-d`：后台分离模式运行。
        *   `-p [宿主机端口]:[容器端口]`：**端口映射**，将容器网络与宿主机绑定。
        *   `-v [宿主机路径]:[容器路径]`：**挂载卷**，实现数据持久化，防止容器删除后数据丢失。
        *   `-e`：传递**环境变量**（如数据库账号密码）。
        *   `--name`：指定容器的唯一自定义名称。
        *   `--restart always/unless-stopped`：设置自动重启策略。
*   **管理状态**：
    *   `docker ps`：查看正在运行的容器（加 `-a` 查看所有容器）。
    *   `docker stop/start [ID或名称]`：停止或重新启动现有容器。
    *   `docker rm -f [ID或名称]`：强制删除正在运行的容器。
*   **调试与检查**：
    *   `docker logs -f [ID]`：滚动查看容器日志。
    *   `docker inspect [ID]`：查看容器详细配置信息（可将复杂信息交给 AI 分析）。
    *   `docker exec -it [ID] [命令]`：在运行中的容器内部执行命令。若想获得交互式终端，命令通常为 `docker exec -it [ID] /bin/bash` 或 `sh`。

### 五、 自定义镜像 (Dockerfile 实战)
1.  **编写 Dockerfile**：创建一个名为 `Dockerfile` 的文本文件，包含以下指令：
    *   `FROM`：指定基础镜像（如 `python:3.13-slim`）。
    *   `WORKDIR`：设定工作目录。
    *   `COPY`：将本地代码拷贝进镜像。
    *   `RUN`：在构建时执行安装依赖等命令（如 `pip install`）。
    *   `CMD`：容器启动时默认执行的指令。
2.  **构建镜像**：执行 `docker build -t [镜像名:版本号] .`。
3.  **推送镜像**：
    *   执行 `docker login` 登录 Docker Hub。
    *   为镜像打上包含用户名的标签。
    *   执行 `docker push [用户名/镜像名]` 上传分享。

### 六、 Docker 内部网络管理
*   **创建子网**：`docker network create [网络名]`，同一子网内的容器可以通过**容器名**互相访问（DNS 机制），无需记住内部 IP。
*   **Host 模式**：启动时使用 `--network host`，容器将直接共享宿主机的网络空间，无需端口映射。

### 七、 多容器编排 (Docker Compose)
当需要同时管理前端、后端和数据库等多个模块时，使用 **Docker Compose**：
1.  **创建 yml 文件**：编写 `docker-compose.yml`，在 `services` 下定义各个容器的镜像、端口、环境变量及依赖关系 (`depends_on`)。
2.  **一键启动**：执行 `docker-compose up -d`，系统会自动创建网络并按顺序启动所有容器。
3.  **停止与清理**：`docker-compose stop` 仅停止容器；`docker-compose down` 则会停止并删除容器及相关网络。
4.  **指定文件**：若文件名不标准，可使用 `-f` 参数指定，如 `docker-compose -f [文件名] up`。

以下是提取的具体操作步骤：

### **一、 平台注册与初始设置**
1.  **访问与登录**：访问 `rcloud.cloud`，点击 **Get Started**。
2.  **账号要求**：推荐使用 **GitHub** 登录。若 GitHub 账号注册超过 **180 天**，每月可获得 **5 美元**的免费赠送额度（足以部署 2-3 个应用）。
3.  **区域选择**：登录后，建议区域选择 **Singapore（新加坡）** 并设置工作区名称。

### **二、 使用应用商店一键部署（以 Lobe Chat 为例）**
1.  **选择应用**：在 **App Store** 中找到 **Lobe Chat** 模板。
2.  **配置 API 密钥**：
    *   **DeepSeek 接入**：在 DeepSeek 开放平台创建 API Key，将其粘贴至配置项的 OpenAI 密钥位置（两者完全兼容）。
    *   **API 地址**：填入 DeepSeek 的接口地址（以 `/v1` 结尾）。
    *   **模型与密码**：填写模型名称（如 `deepseek-chat`）并设置网站访问密码。
3.  **部署与访问**：点击右上角 **Deploy**，在 **Network** 详情页等待系统自动分配公网域名（约需 5-60 分钟）。
4.  **配置更多模型（如 Gemini）**：
    *   在 Lobe Chat 设置中找到“语言模型”。
    *   访问 **Google AI Studio**，在海外上网环境下获取 Gemini 的 API Key。
    *   将 Key 填入 Lobe Chat 并开启 Gemini 选项即可在境内直连使用。

### **三、 配置自定义域名（Cloudflare 托管）**
1.  **设置子域名**：在 Cloudflare 的 DNS 记录中添加 **CNAME** 记录，名称自定义（如 `lob`），目标填写爪子平台分配的原始地址，并将代理状态（小云朵）**关闭**。
2.  **绑定域名**：在爪子平台的应用管理中点击 **Add Port**，选择 **Custom Domain**，填入刚才配置的完整域名并点击 **Confirm**。
3.  **生效**：更新配置后，约需等待 3 小时左右域名即可完成解析。

### **四、 自定义部署（以 AList 网盘为例）**
1.  **创建应用**：在 **App Launch** 中点击 **Create App**。
2.  **镜像配置**：
    *   **镜像库**：填写 AList 官方镜像路径（如 `xhofe/alist:latest`）。
    *   **资源限制**：根据需要调高内存与 CPU 占用。
3.  **网络与环境变量**：
    *   **端口映射**：填入容器内端口（如 `5244`），外部端口由系统自动分配。
    *   **变量设置**：参考 Docker 命令参数，添加容器用户权限等环境变量。
4.  **存储挂载**：添加一个 1GB 左右的存储卷，填写容器内的挂载路径以实现数据持久化。
5.  **初始登录**：部署成功后，查看 **Logs（日志）** 获取默认管理员用户名和密码。

### **五、 部署 AI 工作流（N8N）**
1.  **一键部署**：从 App Store 直接点击 **Deploy** 部署 N8N，无需额外环境变量。
2.  **流程搭建**：进入 N8N 页面完成邮箱注册，通过拖拽节点创建 AI 工作流（例如：对话触发 -> DeepSeek 模型处理 -> 结果保存为文件）。

### **六、 数据库与对象存储**
1.  **数据库部署**：点击 **Database > Create**，选择 **PostgreSQL** 类型并调整配置。
    *   *提示*：若新加坡节点报错，可尝试切换到 **Germany（德国）** 节点创建。
    *   **远程连接**：使用 **DBeaver** 等工具，填入公网 Host、端口和密码即可连通。
2.  **对象存储**：功能类似于 Cloudflare R2，点击 **Create** 创建存储桶并设置权限（如只读）。
    *   **文件分发**：上传文件后可获得公网链接，直接用于浏览器查看或作为项目远程资源。

根据提供的视频来源，该视频详细介绍了 Agent Skills（一种基于开放标准的智能体技能包）的原理及实战操作。以下是提取的具体操作步骤：

### 一、 安装与环境配置
1.  **安装 Claude Code**：
    *   访问 Claude Code 官网并复制**安装命令**。
    *   在桌面右键打开终端，粘贴命令并执行。
    *   **登录方式**：
        *   **官方账户**：输入 `claude login` 登录官网账户直接使用。
        *   **接入其他模型**：在 `C:\Users\用户名\.claude`（Windows）或相应路径（Mac）下新建 `settings.json` 文件。
        *   将模型配置信息（如智谱 AI 的 API Token）粘贴进 JSON 文件并保存，并在 `.claude.json` 中添加相应配置以跳过登录。

### 二、 创作自定义 Skill (以“字幕转 MD”为例)
1.  **建立目录结构**：
    *   新建工程文件夹（如 `skill-project`）。
    *   在工程内依次新建文件夹：`.claude` -> `skills`。
    *   在 `skills` 文件夹下为具体技能创建子文件夹（如 `subtitle-to-md`）。
2.  **创建定义文件**：
    *   在技能子文件夹内新建文件，命名为 **`SKILL.md`**（注意大小写）。
3.  **编写内容**：
    *   **元数据 (Metadata)**：用六个横杠 `------` 包裹，填写技能名称和描述。描述需写清 AI 调用该技能的时机。
    *   **指令 (Instructions)**：使用 Markdown 格式详细描述任务流程（如“保留所有文字”、“插入截图占位符”等）。

### 三、 运行与测试 Skill
1.  **启动工具**：在项目根目录打开终端，输入 `claude` 启动。
2.  **查看技能**：输入命令 `/skills` 确认自定义技能已生效。
3.  **触发任务**：将待处理文件（如 SRT 字幕文件）拖入终端并回车。
4.  **确认调用**：当 AI 询问是否使用该技能处理时，选择 **yes**。AI 会根据指令逐步完成转换并生成结果文件。

### 四、 技能的进阶操作
1.  **全向/全局配置**：将编写好的技能文件夹复制到用户目录下的 `.claude/skills` 目录中，即可在所有项目中生效。
2.  **迁移至其他工具 (如 Codex)**：
    *   在用户目录下的 `.codex` 文件夹中修改 `cfg` 配置文件，开启 `skills` 实验性功能。
    *   将项目中的 `.claude` 文件夹重命名为 **`.codex`** 即可完成迁移。
3.  **使用社区现有技能**：从 GitHub（如 `claude-skills` 项目）下载源码，直接将所需的技能文件夹拖拽到你项目的 `skills` 目录中即可使用。

### 五、 资源层 (Resources) 的高级应用
1.  **组织资源目录**：在技能文件夹下建议创建三个子文件夹：`scripts`（脚本）、`references`（文档）、`assets`（图片等资源）。
2.  **调用外部脚本**：
    *   在 `scripts` 文件夹放入 Python 脚本（如调用 FFmpeg 进行视频截图）。
    *   在 `SKILL.md` 的指令中加入后续处理步骤，要求 AI 在生成文档后调用该 Python 脚本。
3.  **利用参考文档学习风格**：
    *   在 `references` 文件夹放入范文。
    *   在指令中设定逻辑：若处理相关素材，需先读取 `references` 目录下的文档进行风格学习。

### 六、 Skills 与 MCP 的配合使用
1.  **配置 MCP Server**：
    *   以 GitHub MCP 为例，在项目目录执行安装命令。
    *   在 GitHub 页面创建 **API Key**（需开启仓库读写权限），并将其填入安装命令中。
2.  **整合指令**：在 `SKILL.md` 的指令中明确要求 AI 调用特定的 MCP 工具（如 `create-repository` 或 `create-update-file`），从而实现“Skills 负责提示词，MCP 负责具体工具调用”的自动化流程。

以下是视频内容的要点总结：

### 一、 MCP 的核心概念与配置
*   **概念定义**：MCP 是 AI 的**标准化工具箱**，允许 AI 与外部系统对话并代替人类执行操作。
*   **配置工具**：
    *   **Claude Code**：通过命令行 `claudemcp add` 进行配置，支持项目级或用户级生效。
    *   **Codex**：通过编辑配置文件（Windows 为 `cfg.toml`）进行配置，涉及 `npx` 启动命令或远程 URL 的设置。
*   **管理建议**：MCP 不是越多越好，因为会占用上下文，建议“用哪个配哪个”。

### 二、 15 个核心 MCP 及其功能分类
视频将好用的 MCP 工具归纳为以下几类功能：

1.  **浏览器与前端开发**：
    *   **Chrome DevTools MCP (Google 官方)**：让 AI 具备操作浏览器、自动搜索、读取控制台报错并精准修复代码的能力。
    *   **Figma MCP**：通过读取 Figma 设计稿链接，AI 能一句话生成高度还原的网页代码。
    *   **Shadcn/ui MCP**：自动查找并导入高质量的前端 UI 组件。

2.  **后端与数据库管理**：
    *   **Neon MCP**：为 AI 提供免费的云端 PostgreSQL 数据库，实现自动建表和数据存储。
    *   **Supabase MCP**：集成了用户认证、存储等功能的完整后端基础设施，AI 可自动填写 API 密钥等环境变量。

3.  **知识库与文档增强**：
    *   **Context.ai (Cex7) MCP**：补充 AI 截断日期之后的最新编程知识（如 Python 3.14 新特性）。

    *   **Stripe MCP**：为产品快速接入支付能力，AI 可自动获取商品信息并编写支付逻辑。
    *   **Replicate MCP**：通过 API 为产品原型自动生成 AI 配图。

5.  **部署、安全与自定义**：
    *   **Vercel & H1 Pages MCP**：实现项目的一键部署上线。
    *   **Cloudflare MCP**：提供部署、日志监控、流量查询等多种 API 功能。
    *   **Semgrep MCP**：对代码进行静态安全扫描，及时发现漏洞。
    *   **自定义 MCP SDK**：通过官方提供的 Python 等语言 SDK，开发者可以创建专属于自己的 MCP Server。
