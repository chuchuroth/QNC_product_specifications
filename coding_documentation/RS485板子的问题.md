# Hardware Debugging Guide - THVD1410 RS485 Transceiver Issue

## Summary of Investigation
Based on `diagnostic_report.md`, we have confirmed:

### ✅ What's Working
- **Gripper**: Confirmed working with USB-to-RS485 adapter (tested with PC)  
- **Gripper Configuration**: Slave ID and baudrate are correct  
- **Software**: All Modbus framing, CRC, and communication logic is correct  
- **GPIO Control**: GPIO17/18 can be controlled properly  
- **UART**: UART0 transmission is working  

### ❌ What's Not Working
- **RS485 Loopback Failed**: Even with J4 Pin2/Pin3 shorted together  
- **No Gripper Response**: Through the communication module  

## Root Cause
The **THVD1410 RS485 transceiver chip** on the communication module is not functioning properly.

## THVD1410 Configuration
According to `communication_module_wiring.md`:

### J2 Connector (RS485 Interface to CM5)
- **Pin 1: Modbus_RX** → Connect to CM5 UART RX  
- **Pin 2: floating** → Don't connect  
- **Pin 3: Modbus_nRE** → Connect to CM5 GPIO (Receiver Enable, active LOW)  
- **Pin 4: floating** → Don't connect  
- **Pin 5: Modbus_DE** → Connect to CM5 GPIO (Driver Enable, active HIGH)
