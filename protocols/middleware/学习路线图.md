## 🌐 第二部分：网络 & 系统基础

### 🕸 UDP / TCP

你必须理解：

| 概念        | 说明      | 类比        |
| --------- | ------- | --------- |
| TCP       | 可靠、慢一些  | 寄快递签收     |
| UDP       | 快、不保证送达 | 群发消息可能漏一条 |
| Multicast | 一次发给多设备 | 微信群广播     |

入门视频：B站 TCP/UDP 网络开发

💡练习
用C++写一个UDP客户端/服务器程序

---

### ⏱ **实时系统（RTOS、QNX、Embedded Linux）**

最低限度你要知道：

* RTOS：保证“按时间执行”
* Linux：功能全、速度快，但没硬实时
* QNX：车厂、飞机用的超稳定 OS

🧠理解重点：

> 机器人/车辆系统对“延迟”非常敏感

### 📡 DDS / FastDDS

目标：理解 ROS2 底层通信

学会概念：

| 名词                     | 意思             |
| ---------------------- | -------------- |
| Publisher / Subscriber | 发布 / 订阅数据      |
| Topic                  | 频道             |
| QoS                    | 服务质量（可靠性、延迟控制） |
| RTPS                   | DDS的通信协议       |

**FastDDS入门方式**

* 理解 QoS（可靠性、历史深度、延迟优先级）
* 跑 FastDDS examples
* 改 topic / QoS 参数，看性能变化

🧠记住：

> DDS 就是“机器人/自动驾驶的数据传输高速路”

## 🔍 第五部分：调试 & 性能优化

必须掌握工具：

| 工具                    | 用途     |
| --------------------- | ------ |
| `ros2 topic hz`       | 查看频率   |
| `ros2 topic bw`       | 查看带宽   |
| `htop` / `top`        | CPU 监控 |
| `tcpdump` / Wireshark | 抓包分析   |
| `perf`                | 性能分析   |

重点理解：

* 延迟 vs 吞吐
* QoS 配置对系统影响
* 多线程与锁
