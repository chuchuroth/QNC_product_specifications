### 概述
该文档详细描述了 QNC 项目第二阶段（Phase 2），重点内容包括规划功能、总体结构、组件接口、连接方式、潜在挑战以及会议附录。它基于第一阶段的 QNC 模块概念，重点强调与机器人、摄像头、工具的集成，以及支持可分离（ detachable ）操作的电源管理。

### 规划功能（Planned Features）
- **电源与电池**：连接时为末端设备供电并同时为电池充电；分离后自动切换至 UPS 电池模式。电池续航 10-15 分钟，配备电量显示。
- **夹爪功能**：支持多种不同协议。
- **控制接口**：内置屏幕或外部 App 用于控制。
- **AI 功能**：包含视觉和语言能力。
- **物理规格**：可触及表面温度低于 40°C；目标重量 0.5–1.0 kg；从法兰突出距离尽量短。
- **网络**：可能引入汽车以太网（Automotive Ethernet）。

### 总体结构与组件
系统采用模块化设计，主要接口如下：

- **QNC ↔ Module 01 U-Phy Chip**
  - 功能：ModbusRTU、IO-Link、可能包含 CC-Link 和 EIP
  - 接口：内部 API / Driver
  - 示例报文：ModbusRTU 与 IO-Link 的报文格式 TBD
  - 参考：U-Phy Chip 文档（https://docs.rt-labs.com/u-phy/）；IO-Link 主栈 + IODD 解析（示例夹爪型号：LWR50L-02-00002-A）

- **QNC ↔ Robot**
  - 功能：通用功能 API（ON/OFF 等）、法兰连接时为电池充电、位姿跟踪、物体跟踪、夹爪 API、可能支持无线通信和摄像头透传
  - 接口：FastDDS（双向）

- **QNC ↔ Camera D435i**
  - 功能：启动/停止采集、启动/停止位姿跟踪、通过标记转换到机器人坐标系、物体跟踪（AI 功能），可能集成 Jetson Nano
  - 接口：内部 API / Driver

- **QNC ↔ 内置屏幕 / 外部 App**
  - 功能：切换设备模式/运动、通过 NFC 或 Bluetooth 配对、选择夹爪功能、选择摄像头功能、可能通过外部 App 接入麦克风

### 连接方式
- **Robot X → QNC 法兰**：标准化电缆针脚连接器（Neura 8针/12针 或 IO-Link 4针 M12）。必需引脚：VCC、GND、RS485+（半双工）、RS485-（半双工），其余 TBD。
- **QNC 法兰 → QNC 工具**：弹簧加载引脚（参考 Zimmer Match 设计）。引脚要求同上，必须具备短路保护和防误插功能。
- **QNC 工具 → 实际工具/设备**：采用工具制造商原厂完整安装方案 + 标准化连接器（如 OnRobot 的 M8 8针）。几乎肯定需要转接头。

### 潜在挑战 / 注意事项
- 电池尺寸限制
- 电池与固定电源之间的无缝切换（避免供电中断）
- 散热管理：过热会导致 D435i 深度跟踪性能下降，可能需要冷却方案
- 在 NetX90 上实现 ModbusRTU
- 确认精确的电源需求和必需引脚数量

### 概述
该文档概述了 QNC 模块的概念规划和路线图，该模块是一个 24V 供电的组件，基于 NXP i.MX8 PLUS MCU 的嵌入式 Linux 系统。它具有多个外围接口和一个标准化连接器，用于连接末端模块，如机器人夹爪或传感器。一个可选的附加模块允许使用电池供电操作，而不是通常的电缆供电。

### 要求
QNC 模块必须满足以下关键规格：
- 电源：夹爪大约 1A @ 24V，加上其自身功耗 (C-REQ0001)。
- 接口：ModbusRTU、EtherCAT (仅 Class B)、WIFI 5G、ModbusTCP、RS485、RS232、SPI、I2C、以太网、CAN-FD、一个 24V 继电器输出、24V 输入、4 个 3.3V GPIO 输出，以及 4 个 3.3V GPIO 输入 (C-REQ0002)。
- 电池操作：以 24V 充电，并在断开连接时继续运行至少 10 分钟 (C-REQ0003)。
- 软件：灵活，支持轻松实现不同的应用 (C-REQ0004)。
- 尺寸：小于 5 cm × 3 cm (C-REQ0005)。
- 外围设备：能够驱动各种外围设备，如 3D 摄像头、夹爪和真空泵 (C-REQ0006)。
- 成本：PCB 成本低于 €40 (C-REQ0007)。

### 概念和硬件架构
设计围绕 i.MX 8M Plus MCU 连接各种外围设备展开。考虑了现成模块（如 Toradex 的模块），但由于在超过 1,000 单位的大批量时采购成本较高，尽管可以节省 MCU、eMMC 和 RAM 的开发努力，但最终被拒绝。载板仍需要广泛的外围设备。开发使用 MCU 制造商的评估板作为基础，允许软件以最小修改进行适配。

### 电池模块
分为电子组件和电池组。均衡器和保护电子设备可以集成到电池组中（通过专用连接器），或直接安装在电子组件上。在两种情况下，温度测量必须始终直接在电池组内的电池单元上进行。直接将电池单元焊接到 PCB 上是可能的，但不推荐，因为这会使电池不可更换。理想二极管确保在模块移除时不间断运行，重新连接时电池自动充电，无需用户干预。此外，能量测量和 I2C 总线接口允许完全监控电池模块，包括充电状态、充电状况和温度。

### 安装方案
系统是模块化的，由以下组成：
- 电池模块（包括电池组）。
- QNC 模块。
- 从模块（例如，夹爪、3D 成像设备或其他传感器）。
