# 🤖 Robotics Embedded Systems Development Setup Guide

## 📋 Core Development Environment

### 1. **Primary IDE: VS Code** (Free, Most Flexible)
**Why:** Best ROS 2 integration, massive extension ecosystem, lightweight

**Essential Extensions:**
- **ROS** by Microsoft - ROS/ROS2 support, URDF/launch file syntax
- **C/C++** by Microsoft - IntelliSense, debugging, code navigation
- **Python** by Microsoft - Python development
- **CMake Tools** - CMake project management
- **Remote - SSH** - Remote development on robots/embedded devices
- **Remote - Containers** - Docker development environments
- **GitLens** - Advanced Git integration

**Installation:**
```bash
# Install VS Code
wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -D -o root -g root -m 644 packages.microsoft.gpg /etc/apt/keyrings/packages.microsoft.gpg
sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
sudo apt update
sudo apt install code
```

**ROS 2 Workspace Template:**
- Use the pre-configured template: [vscode_ros2_workspace](https://github.com/athackst/vscode_ros2_workspace)
- Includes debugging, linting, Docker support out of the box

---

### 2. **Alternative IDE: CLion** (Paid, $89/year, Better for C++)
**Why:** Superior C++ refactoring, better CMake integration, professional debugging

**Best for:**
- Complex C++ robotics projects
- Heavy MoveIt2/navigation stack development
- When you need advanced refactoring tools

**Setup for ROS 2:**
- Launch CLion from sourced terminal: `source /opt/ros/jazzy/setup.bash && clion`
- Open workspace via `compile_commands.json` (generated by colcon)
- Configure external tool for `colcon build`

---

### 3. **For STM32/Microcontrollers: STM32CubeIDE**
**Why:** Hardware abstraction layer (HAL) generation, integrated debugger

**Note:** Currently lacks native GitHub Copilot support. Use VS Code with PlatformIO as alternative for AI-assisted embedded dev.

## 🛠️ Essential Development Platforms & Tools

### **ROS 2 Environment**
```bash
# Install ROS 2 Jazzy (latest LTS)
sudo apt install ros-jazzy-desktop-full

# Development tools
sudo apt install python3-colcon-common-extensions
sudo apt install python3-rosdep python3-vcstool

# Workspace setup
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws
colcon build --symlink-install
```

### **Simulation Platforms**

**1. Gazebo Harmonic** (Free, Open Source)
```bash
sudo apt install ros-jazzy-gazebo-ros-pkgs
```

**2. NVIDIA Isaac Sim** (Free, NVIDIA GPU Required)
- Download from NVIDIA Omniverse
- Best for photorealistic simulation
- GPU-accelerated sensor simulation

sudo snap install webots

## 💡 Pro Tips

### **AI Assistant Usage:**
- **Use Cursor/Copilot for:** Boilerplate ROS nodes, launch files, URDF generation, CMakeLists.txt
- **Write yourself:** Safety-critical control loops, real-time algorithms
- **Always review:** Generated code for embedded/real-time systems

### **Performance:**
- Use `ccache` for faster C++ compilation
- Build with `-DCMAKE_BUILD_TYPE=Release` for production
- Profile with `ros2 topic hz` and `htop`

### **Debugging:**
- Use `rqt_graph` to visualize node connections
- `ros2 topic echo` for message inspection
- GDB with VS Code for C++ debugging
- Set `RCUTILS_LOGGING_SEVERITY=DEBUG` for verbose logging
