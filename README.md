* 先给项目瘦身，把我的文件统一下撤：（准备一套场面话解释），为了不confuse开发工作，现在把过于broad的定义，或太radical “大跃进”式的计划先取消。基于现实情况来制定具体计划：让be负责他的工作，让ma负责他的工作，把maira的东西弄完就close project qnc。既然我的工作无人配合（他们也没这个能力），那我自己去杭州再组建团队，准备工作已经完成，就照着我现在做出来的roadmap继续做，换个项目名字即可。话不投机半句多，没必要，国内有的是能人，何必跟这帮人废话。




* 借用ahmet弄一套思路出来，做一个“杭州分部发展策划”，包括地缘政治风险规避，如何跟当地政府搞好关系，如何争取资源，如何管理整条供应链（先把机器人身上2000-3000零件列出来，挨个分析来源，分析每条供应链上的风险及应对策略）。说服mav，他肯定需要这么一个“掂得清轻重”的人，既懂技术，又懂行业，又能跟现实接轨，要让他知道：我什么事都心里有数，交给我，你放心。
* 其实CU的这个部门的角色定位，在riederich也是处境尴尬，别的business unit都锚定在产品上，合情合理又好理解，唯有computer unit，大部分人的第一反应就是这个部门到底是干啥的，它负责什么产品？如果它参与到别的business unit的工作中，就会导致一个尴尬局面，就是那些已经长期深耕于那个产品的“专家们”，我们无法做到比他们更了解那个产品，因此很难找到工作融合的切入点，而如果专注于某项前沿技术的研究，类似于高校的“实验室”，我们的研发能力显然不足以与高校匹敌（再说我们已经有AI部门，先进技术他们更有权威性）。如果真想另设炉灶，杭州倒是一个不错的选择，在那里有优秀的人才，我们可以轻松组建一个本地化的专业团队，成本还低，既可以极大提高自主性，还可以资源对接，形成一股后备力量，对riederich的布局起到支持、完善和保护作用，又可以提高整体的研发能力。对于杭州的规划，我做了一些思考，主要从地缘政治风险的规避等几个角度出发。我认为大规模地在那里增设厂房，增加生产规模，不是一个明智的选择，因为树大招风，这容易引起各方注意，还有关税的限制。从成本控制的角度，以前吸引外商是中国的廉价劳动力，而现在情况有些不同，自动化程度和AI应用技术的提升，人力成本可以通过很多种方式控制，并不一定需要大量劳动力才能运转一个工厂。再者，万一发生社会动荡之类的无法预测的事件，厂房设备相对于几个人来说更难撤离，硬件设施投入越大，风险就越大。更明智的做法是设立一个小型的办公室或实验室，组建一个小型团队，这样做也不显眼，目的就是方便整合国内的资源，管理供应商等，顺便保证信息通畅，也是提升研发能力。有些solution是经过验证的，是一百年间外资们走通了的路径，即便是目前的地缘政治形势跟一百年前有所不同，但底层逻辑是相通的，有变化的部分也有不变的部分，在这点上，我们可以借鉴Bosch的做法，他们的决策必定是经过专门的专家们经过一系列完整的分析政策风险，我们照着做就错不了。





"我发现我们目前只有寥寥几家国内供应商，这个数量太低了，国内的资源一定要好好利用起来，得有专人去研究国内的下游供应商和资源管理。"



"再说，社会动荡既是风险，同时也是契机，虽然趁人之危是不道德的，但倘若有一天，国内的局势真乱了，那么也就是执行某些非常规的商业竞争手段的最佳时机。"
